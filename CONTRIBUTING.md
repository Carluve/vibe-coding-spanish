# Cómo Contribuir

¡Gracias por querer contribuir a este repositorio! Esta guía te explica cómo hacerlo correctamente.

## Tipos de contribuciones bienvenidas

- **Nuevos repositorios oficiales** de `github.com/cloudflare/*` que aún no estén listados
- **Correcciones** de enlaces rotos o descripciones incorrectas
- **Guías en español** nuevas o mejoradas
- **Recursos comunitarios** de calidad (tutoriales, vídeos, artículos)

## Proceso para contribuir

1. **Fork** este repositorio
2. Crea una rama descriptiva:
   ```bash
   git checkout -b añadir-guia-workers-d1
   ```
3. Realiza tus cambios en `README.md` u otros archivos relevantes
4. Asegúrate de que los enlaces funcionan correctamente
5. Haz commit con un mensaje claro:
   ```bash
   git commit -m "Añade guía de Workers con D1 en nivel intermedio"
   ```
6. Haz push y abre un **Pull Request**

## Criterios de calidad

### Para repositorios
- Deben pertenecer a la organización oficial `github.com/cloudflare`
- O ser recursos comunitarios con alto nivel de mantenimiento y uso
- Incluir descripción clara en español

### Para guías
- Deben ser precisas y estar actualizadas
- Código de ejemplo debe ser funcional
- Enlazar a la documentación oficial correspondiente

### Para enlaces
- Verificar que el enlace está activo antes de añadirlo
- Preferir enlaces a `developers.cloudflare.com` sobre terceros

## Estilo

- Usa español de España (o neutro) en descripciones y guías
- Mantén el formato de tablas Markdown existente
- Los bloques de código deben especificar el lenguaje (`bash`, `typescript`, `javascript`, etc.)

## Reportar problemas

Si encuentras un enlace roto, información desactualizada o cualquier error, puedes:
- Abrir un **Issue** describiendo el problema
- Enviar directamente un **Pull Request** con la corrección
